# プロジェクトルール

## 要件定義の参照

- 実装前に必ず `documents/` ディレクトリ以下の要件定義書（`index.md`）と実装計画（`plan.md`）を確認すること
- 要件定義書に記載されている仕様に従って実装すること
- 実装計画に沿って段階的に実装を進めること

## 要件外の実装について

- 要件定義書や実装計画に記載されていない機能・実装を追加する場合は、**必ずユーザーの許可を得てから実装すること**
- 実装の提案をする場合は、その理由と必要性を明確に説明すること
- ユーザーが明示的に許可するまで、要件外の実装は行わないこと

## 実装方針

- 各ステップで動作確認を行いながら段階的に実装を進める
- 依存関係を考慮し、下位レイヤー（utils, entities）から実装する
- TypeScript の型安全性を活用する
- エラーハンドリングは統一形式で実装する

## コード生成・仕様書生成時のルール

- 仕様を書く前に、関連するソースコードを必ず確認すること
- 推測や一般論ではなく、実際のコードに基づいて記述すること
- 現在の実装で実現していることのみを仕様として記載すること
- 不確かな内容は記載しないこと
